License Reclaim Playbook: Compare, Verify, Revoke

Published February 14, 2026 • 8 minute read

This playbook gives IT and procurement teams a repeatable model for reducing software license waste without creating risky removals or weak audit records.

When to Run This Playbook

  • Your renewal date is within the next 30 to 90 days
  • You have license assignment exports and directory exports available
  • Your team needs documented evidence for reclaim decisions

Step 1: Prepare Two Source Files

Export one CSV from your license source and one CSV from Active Directory. Make sure both include at least one shared user key such as email address.

Step 2: Run Comparison

Use NyLi Assets comparison to identify matched users, users only in license files, users only in AD, and field-level differences. This is your initial reclaim candidate pool.

Step 3: Launch Verification

Send verification requests to flagged users using one-click response links. Track status changes across pending, confirmed, and revoked so your team can move quickly but safely.

Step 4: Prioritize Reclaims

Focus first on revoked responses and long-pending requests with no activity. Use project-level status views and run history to prepare an approval-ready reclaim list.

Step 5: Export Evidence

Export run results and audit records with timestamps and response history for procurement, security, and compliance stakeholders.

Key Takeaway

Teams that standardize compare, verify, and evidence export cycles reclaim faster with less operational risk.
